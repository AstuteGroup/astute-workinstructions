Pricing Models (two distinct, do not conflate)

Broker-to-broker (Stock RFQ, Suggested Resale):
~20-30% of franchise best price. <$1→20% · $1-10→25% · $10+→30%. Broker sales override if market clears differently.

Direct customer (Quick Quote):
Margin/GP-based. 15% min margin, $250 min GP/line. 6-tier suggested resale hierarchy (same-cust PPV/shortage/losing CQ → other-cust → target → 30% fallback).

Major changes since v1 (March 2026)

  • iDempiere REST API write-back is live in PRODUCTION (172.31.7.239). Tsunami User role 1000004. Six writers ship: rfq-writer, offer-writeback, vq-writer, cq-writer, record-updater, api-result-writer. FDW write-back approach superseded.
  • RFQ Loading via AI ships — AI extraction from customer emails/attachments, load-first/enrich-later architecture. 9,003-line Honeywell PPV loaded at ~20 lines/s.
  • HTS / ECCN Backfill workflow added — RFQ-scoped PATCH of chuboe_vq_line via DigiKey + Mouser.
  • mfr-equivalence.js now standard for all "same company?" checks. 200+ aliases + acquisitions chain (Linear→ADI, Atmel→Microchip, Fairchild→Onsemi, etc.).
  • mpn-match.js fixes the "recommendation fallback" bug — all 10 distributor parsers pre-2026-04-14 silently surfaced products[0] as the searched part on miss. Now: exact + variant tiers, optional MFR veto, return found=false on miss.
  • Bean-callout traps documented — chuboe_offer_line CPC dedup silently merges MPNs; Chuboe_CPC non-updateable after POST; workarounds shipped (per-CPC anchor + offer_line_mpn sub-rows).
  • Cron jobs — Inventory Cleanup (Mon 6am) · LAM Kitting Reorder · API Queue Worker (*/30). Operator emailed on exhaustion.
  • Heilind scraping ruled out (Imperva WAF tarpits headless Chromium). Pursuing EDI 832/846 as durable path.
